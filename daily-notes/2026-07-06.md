# Day 14 Project Notes - 2026-07-06

## Summary

Day 14 focused on building the login page UI, connecting it toward the backend login API, and enabling frontend-to-backend API calls.

## Backend Changes

- `Backend/package.json`
  - Added the `cors` dependency.

- `Backend/index.js`
  - Imported `cors`.
  - Added `app.use(cors())` before JSON parsing and route registration.
  - This allows the Vite frontend to call the backend API from a different origin during development.

## Frontend Changes

- `Frontend/src/App.jsx`
  - Enabled `<Toaster position='bottom-center'/>` so React Hot Toast messages can display in the app.

- `Frontend/package.json`
  - Added the `axios` dependency.

- `Frontend/package-lock.json`
  - Updated lockfile entries for `axios` and its dependency tree.

- `Frontend/src/App.css`
  - Replaced the old `.headline` style with Tailwind CSS import and custom theme colors.
  - Added theme colors:
    - `primary`: `#d0e7e6`
    - `secondary`: `#293681`
    - `accent`: `#4274D9`

- `Frontend/src/main.jsx`
  - Removed the trailing comma after the React `<StrictMode>` render tree.
  - No functional behavior change expected from this edit.

- `Frontend/src/pages/loginPage.jsx`
  - Replaced the simple placeholder login page with a full login form UI.
  - Added local React state for `email` and `password`.
  - Added `axios.post("http://localhost:3000/users/login", ...)` inside `handleLogin`.
  - Added toast error handling for failed login attempts.
  - Added email and password inputs.
  - Added email and password icons using `MdEmail` and `MdKey`.
  - Added links for forgot password and signup.
  - Changed the Sign in button from `onChange` to `onClick`.
  - Added a temporary success toast that shows the entered email and password.
  - Removed input value console logs from the email and password fields.
  - Added a Google icon with `FcGoogle` and improved the Google sign-in button layout.
  - Added background image styling and a blurred login panel.

- `Frontend/public/bg_image.png`
  - Added the login page background image asset.

## Watch Items

- In `Frontend/src/pages/loginPage.jsx`, the Axios response is not stored, but `console.log(res)` is still used. This will cause `res is not defined` after a successful request.
- The temporary success toast currently displays the password; remove this before continuing with real login testing.
- The login background image is referenced as `bg-[url('bg_image.png')]`; with Vite public assets, `/bg_image.png` is usually safer.
- The forgot-password link points to `/forget-password`, but no matching route is currently registered in `App.jsx`.
- The Google sign-in button is UI-only right now and has no login logic.
- The login request may return `404` when the email is not found in MongoDB; this is different from a CORS error.
